# 基于SpringBoot+Vue的非物质文化遗产展示与交流平台 Spec

## Why

非物质文化遗产是中华民族宝贵的文化财富，需要一个现代化的数字化平台来展示、传播和交易非遗物品，促进非遗文化的传承与发展，同时为传承人提供商业变现渠道。

## What Changes

* 构建完整的前后端分离的Web应用平台

* 实现多角色权限管理系统（普通用户、传承人、管理员）

* 开发非遗展示、文创商品、互动交流等核心功能模块

* 建立完善的用户体系和内容管理体系

## Impact

* 技术栈：SpringBoot 2.x/3.x + Vue 3.x + MySQL 8.x

* 架构模式：前后端分离，RESTful API设计

* 涉及系统：用户系统、非遗展示系统、文创交易系统、内容系统、消息系统

***

## 系统角色说明

### 三种角色

| 角色 | 说明 | 所属端 |
|------|------|--------|
| 普通用户 | 浏览非遗内容、购买文创商品、互动交流 | 前台 |
| 传承人 | 发布非遗项目、管理文创商品、处理订单 | 前台 |
| 管理员 | 审核传承人、管理内容、数据统计 | 后台 |

### 角色转换机制

* 注册时可选择：普通用户 / 传承人
* 已注册的普通用户可在前台申请成为传承人
* 传承人申请需上传资质证明、证书，经管理员审核通过后方可成为传承人
* 传承人申请时需选择自己的非遗类型（剪纸、刺绣、陶艺等）
* 传承人只能管理、发布自己所属类型的内容

## ADDED Requirements

### Requirement: 用户认证与授权系统

系统应提供完整的用户认证与授权功能，支持多角色管理和角色转换。

#### Scenario: 用户注册

* **WHEN** 新用户填写注册信息并选择角色类型（普通用户/传承人）
* **THEN** 系统创建对应角色账户，传承人需等待审核

#### Scenario: 用户登录

* **WHEN** 用户输入正确的用户名和密码
* **THEN** 系统验证身份并返回认证令牌，根据角色跳转对应页面

#### Scenario: 传承人申请

* **WHEN** 普通用户提交传承人申请（包含资质证明、证书、非遗类型）
* **THEN** 系统记录申请并等待管理员审核

#### Scenario: 传承人审核

* **WHEN** 管理员审核传承人申请
* **THEN** 系统更新用户角色并通知申请人审核结果

### Requirement: 非遗展示功能

系统应提供非遗项目的展示、浏览和搜索功能。

#### Scenario: 非遗列表展示

* **WHEN** 用户访问非遗展示页面
* **THEN** 系统展示分类栏（左侧）、项目列表（右侧），包含缩略图、名称、发布者信息

#### Scenario: 分类筛选

* **WHEN** 用户点击左侧分类（剪纸、刺绣、陶艺、皮影、木雕等）
* **THEN** 系统筛选并展示该分类下的所有非遗项目

#### Scenario: 排序功能

* **WHEN** 用户选择排序方式（点击量、收藏数、点赞数）
* **THEN** 系统按升序/降序排列项目列表，显示排序箭头指示

#### Scenario: 搜索功能

* **WHEN** 用户在顶部搜索框输入关键词
* **THEN** 系统返回匹配的非遗项目列表

#### Scenario: 非遗详情查看

* **WHEN** 用户点击非遗项目进入详情页
* **THEN** 系统展示：历史渊源、技艺特点、高清大图、制作视频（在线播放）、评论互动区

#### Scenario: 发布者信息展示

* **WHEN** 查看非遗项目详情
* **THEN** 管理员发布的内容不显示传承人信息；传承人发布的内容显示用户名、头像，点击可跳转传承人主页

#### Scenario: 传承人主页

* **WHEN** 用户点击传承人头像或用户名
* **THEN** 系统跳转到传承人主页，展示该传承人发布的所有非遗内容

## 系统架构设计

### 技术选型

| 层级    | 技术方案                                    |
| ----- | --------------------------------------- |
| 前端框架  | Vue 3.x + Element Plus                  |
| 后端框架  | SpringBoot 2.7.x / 3.x                  |
| 数据库   | MySQL 8.x                               |
| 认证    | JWT Token                               |
| 文件存储  | 本地存储（服务器磁盘）                              |
| API文档 | Swagger / Knife4j                       |

### 数据库设计概要

#### 用户相关表

* `sys_user` - 用户基础信息表
* `sys_role` - 角色表
* `sys_user_role` - 用户角色关联表
* `inheritor_apply` - 传承人申请表（含资质证明、证书、非遗类型）

#### 非遗相关表

* `heritage_item` - 非遗项目表
* `heritage_category` - 非遗分类表（剪纸、刺绣、陶艺、皮影、木雕等）
* `heritage_image` - 项目图片表
* `heritage_video` - 项目视频表

#### 文创相关表

* `product` - 文创商品表
* `product_category` - 商品类型表
* `cart` - 购物车表
* `order` - 订单表
* `order_item` - 订单明细表

#### 内容相关表

* `news` - 资讯表
* `forum_post` - 论坛帖子表
* `message` - 留言表
* `announcement` - 公告表
* `banner` - 轮播图表

#### 互动相关表

* `user_favorite` - 用户收藏表
* `user_like` - 用户点赞表
* `comment` - 评论表
* `consultation` - 咨询表
* `sensitive_word` - 敏感词表

│   ├── mapper/         # 数据访问层

│   ├── dto/            # 数据传输对象
│   ├── vo/             # 视图对象
│   ├── config/         # 配置类
│   ├── security/       # 安全认证
│   ├── exception/      # 异常处理
│   └── util/           # 工具类
└── src/main/resources/
